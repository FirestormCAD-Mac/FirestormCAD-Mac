## 📖 Overview

<a href="#firestormCAD">FirestormCAD</a> serves the professional Mac user who needs genuine CAD capability — not the simplified drawing tools bundled in general-purpose applications but the precision drafting, constraint-based design, and industry-standard file compatibility that professional technical work requires.

The <a href="#firestormCAD">architecture and construction use case</a> applies FirestormCAD to floor plans, elevation drawings, site plans, and construction details — the technical drawing types that communicate building designs to contractors, permit authorities, and structural engineers. Precise dimensioning, layer organization separating different building systems, and DWG compatibility with engineering consultants make FirestormCAD applicable to architectural practice. The <a href="#firestormCAD">mechanical engineering use case</a> uses FirestormCAD for part drawings, assembly diagrams, and manufacturing documentation — creating the dimensioned technical drawings that machinists and fabricators work from to produce physical components.

The <a href="#firestormCAD">Mac-native advantage</a> provides the smooth, responsive interface behavior and system integration that engineers and architects who work primarily on Mac expect — native file dialog integration, Retina display support for sharp drawing rendering, and the performance of a native application rather than a cross-platform tool running a compatibility layer. <a href="#firestormCAD">Professional output quality</a> produces drawings that meet the standards of technical documentation — clean line weights, accurate dimensions, correct annotation formatting, and scaled output suitable for inclusion in professional project packages.</p>

<a href="#firestormCAD">3D modeling capabilities</a> extend beyond 2D drafting to three-dimensional form creation — extruding 2D profiles into 3D solids, boolean operations between 3D bodies, and 3D viewing for design verification before committing to 2D documentation. <a href="#firestormCAD">Template and standards system</a> creates reusable drawing templates with predefined layer structures, dimension styles, text styles, and title blocks for consistent output across all drawings produced in a practice or organization.

<a href="#firestormCAD">Block and symbol library</a> stores reusable drawing elements — standard architectural symbols, mechanical fastener representations, electrical schematic symbols — for insertion into drawings without redrawing common elements repeatedly. <a href="#firestormCAD">Reference file support</a> attaches external drawing files as underlays — viewing referenced drawings as a transparent background layer for coordination and overlay without merging the reference into the active drawing file.</p>

The parametric constraint system is the capability that separates professional CAD tools from advanced drawing applications. When dimensions and geometric relationships are managed by constraints rather than manual placement, design changes propagate correctly through the drawing — moving one wall updates the dimensions, related room sizes, and dependent elements automatically. This constraint-driven behavior is the foundation of professional CAD methodology and the reason that design iteration in a constrained CAD model is faster than in unconstrained drawing tools.

> *"The parametric constraints changed how I approach design iteration. I define the geometric relationships once, and when the client changes a dimension, the related geometry updates. What used to be a manual redrafting task is now a constraint update."* — James R., Technical Designer
